### Transform组件
#### 组件介绍
1. Transform组件用于存储GameObject的position、rotation、Scale、和父子状态。一个GameObject总是会有一个Transform组件。
2. scale属性，在检视面板上拥有一个Proportional标记，选中可将GameObject沿x、y、z轴的缩放成比例的缩放。
#### Transform Script脚本
1. Transform脚本类型支持应用position、rotation、scale等。他们也支持enumerator,这样你可以通过如下代码循环子级:
``` C#
using UnityEngine;

public class Example : MonoBehaviour
{
    // Moves all transform children 10 units upwards!
    void Start()
    {
        foreach (Transform child in transform)
        {
            child.position += Vector3.up * 10.0f;
        }
    }
}
```
##### Method
1. Translate方法
public void Translate(Vector3 translation);  
public void Translate(Vector3 translation, Space relativeTo = Space.Self);  
作用: 在指定的方向和距离上进行平移。  
参数relativeTo如果是省略或者设置为Space.Self，表示按照transform自己的本地轴方向移动,如果是  
Space.World表示按照世界坐标系统进行移动。
``` C#
using UnityEngine;
using System.Collections;

public class ExampleClass : MonoBehaviour
{
    void Update()
    {
        // Move the object forward along its z axis 1 unit/second.
        // 在z轴上移动transform，每秒一个单位距离
        transform.Translate(Vector3.forward * Time.deltaTime);

        // Move the object upward in world space 1 unit/second.
        transform.Translate(Vector3.up * Time.deltaTime, Space.World);
    }
}
```
还有如下声明的方法:
指定分别指定x轴、y轴、z轴的移动
public void Translate(float x, float y, float z);  
public void Translate(float x, float y, float z, Space relativeTo = Space.Self);  
移动transform按照指定方向和距离，同时指定移动是根据哪个transform的本地坐标进行移动。如果为null的话是按照世界坐标进行移动。  
public void Translate(Vector3 translation, Transform relativeTo);  
同样的单独指定x、y、z的版本。  
public void Translate(float x, float y, float z, Transform relativeTo);

### Rigidbody2D组件
2D Sprite的刚体物理组件,将一个对象放置在物理引擎下控制。与标准Rigidobody组件的区别在于，在2D情况下，物体只能在XY面板上移动，并且只能绕垂直于(perpendicular)XY面板的轴进行移动。
1. Rigidbody2D是怎么工作的.  
   当一个物体的Transform组件改变的时候，会更新其他组件，像是渲染或者colliders碰撞器的位置。2D 物理引擎能够移动碰撞器然后让他们互相交互，所以物理引擎需要一个方法使得colliders碰撞器的移动反馈到Transform组件。这个colliders碰撞器的移动和沟通就是Rigidbody2d组件完成的。  
   Rigidbody2D组件覆写Transform并更新他的position/rotation,但是你依然可以自行修改Transform组件来控制Rigidbody2D.这样做可能会造成GameObject互相穿过或者进入，已经无法预料到的移动。  
   任何Collider2D组件添加到同一个GameObject或者子GameObject都会隐式附加到Rigidbody2D.当一个Collider2D碰撞器附加到Rigidbody2D,会和它一起移动。一个碰撞器不应该直接通过Transform或者任何collider碰撞器偏移量进行移动。应该通过Rigidbody2D进行移动。这将会支持最好的表现，并且保证正确的collision碰撞检测.多个Collider2D附加到同一个Rigidbody2D上不会互相碰撞。这表示你可以创建一系列的colliders碰撞器作为一个复合碰撞器，所有的移动和旋转rotation与Rigidbody2D同步。  
   提示:  
   添加一个Rigidbody2D允许一个sprite通过脚本API施加一个力进行物理的、可信的方式移动
#### 设置
1. BodyType设置，会影响到组件上的其他设置。  
   有三种选项，每一种定义一个通用的固定的行为。任何附加到Rigidbody2D的collider2D将会继承这个BodyType.
   1. Dynamic 动态的
   2. Kinematic 力学的
   3. Static 静态的  
   这些选项定义:移动的行为(position 和 rotation).Collider interaction碰撞器交互.注意虽然Rigidbody2D经常用于描述互相之间的碰撞，但他是附加在这些物体上的colliders的碰撞。Rigidbody2D之间是不能进碰撞的，没有碰撞器的话。  
   更改Rigidbody2D的BodyType是一个棘手的过程，内部互相关联的多种属性会立即重新计算，以及所有存在的附加在Rigidbody2D的交互需要被重新计算，在GameObject的下一次FixedUpdate调用中。  
   取决于有多少contacts和Collider 2D附加在Rigidbody上，改变BodyType可能造成多种不同的表现。  
>
    1. Dynamic (动态的)
        Dynamic设计用于在模拟下进行移动，它有完整的属性集合支持它，通过gravity和forces产生影响。一个Dynamic Body会和其他所有类型的Rigidbody2D进行碰撞。是Rigidbody2D的默认类型，因为这是最  通用的用于移动物体的body类型。同时也是最消耗性能的bodyType,因为他的动态特性和与周围所有事物交互的特性。所有的Rigidbody2D属性都能使用在这种bodytype下。  
        不要直接设置一个Dynamic Rigidbody2D的Transform的position或者rotation，模拟改变一个Dynamic Rigidbody2D通过它的velocity,你可以改变直接通过应用一个力forces通过脚本，或者非直接的  通过collisions碰撞和gravity重力
    2. Kinematic (力学的)
        Kinematic Rigidbody2D也是设计用于模拟下进行移动，但仅限于非常明确的用户控制之下。Dynamic Rigidbody 2D是被gravity和forces影响的，但kinematic Rigidbody 2D不是。因为如此，  它很快并且对于系统资源相较于Dynamic Rigidbody 2D更少。Kinematic Rigidbody 2D设计来通过Rigidbody2D.MovePosition或者Rigidbody2D.MoveRotation来显式重定位的。  
        使用物理查询进行collisions检测，以及脚本决定如何已经向哪里Rigidbody 2D应该移动。  
        一个Kinematic Rigidbody 2D依旧通过velocity进行移动，但是velocity不被forces和gravity影响。Kinematic Rigidbody 2D不与另外的Kinematic Rigidbody 2D或者Static Rigidbody 2D进行碰撞；它只与Dynamic Rigidbody 2D进行碰撞。类似于 Static Rigidbody 2D，在碰撞中,一个Kinematic Rigidbody 2D的特性像是一个无法移动的物体。Mass 相关的属性无法用于这个Body Type
    3. Static (静态的)
        Static Rigidbody 2D被设计来不在模拟下进行移动；任何与它产生的碰撞，Static Rigidbody 2D表现的像一个无法移动的对象。它也是最少得资源使用的body type.Static Rigidbody 2D只与  Dynamic Rigidbody 2D进行碰撞。两个Static Rigidbody 2D碰撞是不支持的，因为Static Rigidbody 2D不是设计用于移动的。这种Body Type可使用的属性很有限。
>  
#### 属性
    Material: 用于指定一个通用与所有附加到这个父Rigidbody 2D的collider 2D碰撞器的材质。  如果collider 2D有自己指定材质，就使用它自己的，没有设置或者附加到的Rigidbody 2D没有Material设置，默认为None.  
    Simulated: 设置是否启用模拟，如果不想这个Rigidbody 2D与其他Rigidbody 2D进行交互，将这个选项设置为disable。  
    Linear Drag: 影响位移的阻力系数  
    Angular Drag: 影响旋转的阻力系数  
    Gravity Scale: 设置刚体受重力影响的系数  
    Collision Detection: 定义两个collisions之间如何进行碰撞检测。  
    1. Discrete: 这个设置下，拥有Rigidbody 2D和Collider 2D的GameObject在物理更新期间可能会重叠在一起或者直接穿过去。如果速度够快，碰撞可能在新的位置发生。
    2. Continuous: 这个设置下，拥有Rigidbody 2D和Collider 2D的GameObject在物理更新期间不会重叠在一起或者直接穿过去。它会计算两个碰撞器的第一个接触点，然后将GameObject移动到那儿。会使用更多的CPU计算。  
    Interpolate: 定义GameObject的移动在物理更新之间是怎么插值的。  
        1. 不引用运动平滑设置
        2. 基于GameObject前一帧的运动平滑
        3. 基于估算GameObject在一下帧位置的运动平滑计算。
    Constraints: 定义任何Rigidbody 2D的运动限制。
